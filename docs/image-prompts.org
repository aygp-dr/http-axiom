** Ollama CLI Usage

Ollama *does* support image generation via ~ollama run~ for models with
the ~image~ capability. The ~x/z-image-turbo:latest~ model (10.3B params,
FP8, 12 GB) shows a "Generating" progress bar when invoked.

*Known issue:* On this machine, the MLX dynamic library is not available
(ollama 0.16.3 searches ~/opt/homebrew/Cellar/ollama/0.16.3/bin~ and
the local build directory). Without MLX acceleration, the model falls
back to CPU inference, which is prohibitively slow for a 10.3B diffusion
model (stuck at 0/9 steps after 60 seconds).

#+begin_src bash
# Verify model capabilities
ollama show x/z-image-turbo:latest
# Should show: Capabilities: image

# Generate image (requires MLX or GPU acceleration)
ollama run x/z-image-turbo:latest "Generate an image: [paste prompt here]"
# Output: generates image inline in terminal (iTerm2/kitty) or saves to file

# Fix MLX acceleration (if on Apple Silicon):
# 1. Ensure ollama is updated: brew upgrade ollama
# 2. Check MLX library path in ollama logs
# 3. Alternatively, use the ollama API:
curl -X POST http://localhost:11434/api/generate \
  -d '{"model": "x/z-image-turbo:latest", "prompt": "..."}'

# Alternative local tools for image generation:
# - ComfyUI with Flux models
# - Automatic1111 / Forge with SD models
# - diffusers Python library with local GPU
#+end_src
