* What to do differently in the rebuild

** 1. Use flag.FlagSet (stdlib, zero deps)

The hand-written parsing caused 5 bugs (--json position, -V/-v collision,
documented-but-unparsed flags, -n ambiguity, help only in first position).
=flag.FlagSet= per command prevents all of these while staying zero-dep.

** 2. Types first, features second

Commit order should be:
1. =internal/request/request.go= (domain type)
2. =internal/predicate/types.go= + =runner.go= (framework)
3. Per-group predicate files (empty, with NamedPred registration)
4. =internal/generator= + =internal/mutation= + =internal/relevance=
5. =internal/executor= + =internal/oracle=
6. =internal/cli/= (one file per command)
7. =internal/output/= (presentation tier)
8. =main.go= (~50 lines: parse subcommand, dispatch to cli.RunFoo)
9. Predicate implementations (per-group agents, zero conflicts)
10. =cmd/haxgoat/= (test target)

** 3. aq as first-class in agent workflow

#+begin_src bash
# Before any agent starts:
aq announce -c <bead-id> -f "<files>" --plan "<what I'm doing>"

# Before editing a file:
aq check -f "<file>" --plan "<function I'm adding>"
# If conflict detected: coordinate or queue

# After commit:
aq announce -c <bead-id> --phase "proof" -f "<files>"
#+end_src

Hook this into Claude Code hooks so it's automatic:
#+begin_src json
{
  "hooks": {
    "Edit": {
      "pre": "aq check -f ${file} || echo 'CONFLICT WARNING'"
    },
    "Bash[git commit]": {
      "post": "aq announce --phase proof"
    }
  }
}
#+end_src

** 4. Per-feature test files from day one

Instead of =predicate_test.go= (conflicts on every merge):
- =headers_test.go= (headers predicates only)
- =methods_test.go= (methods predicates only)
- =crossorigin_test.go= etc.

** 5. sb for worktree management

#+begin_src bash
sb init              # setup worktrees/
sb add agent-csp     # create worktree for CSP agent
sb add agent-hsts    # create worktree for HSTS agent
sb audit             # verify all worktrees under worktrees/
sb list              # show status
#+end_src

** 6. cprr for every predicate

Each predicate is a conjecture:
#+begin_src bash
cprr add "CSP unsafe-inline + unsafe-eval nullifies protection" \
  --hypothesis "parseCSPDirectives → fail on both present in script-src"
cprr add "HSTS max-age=0 disables protection per RFC 6797 §6.1.1" \
  --hypothesis "parseHSTSMaxAge → fail on 0, warn on <31536000"
#+end_src

Advance through: open → testing → confirmed (with evidence from haxgoat).

** 7. bd dependency chains prevent wrong-order work

#+begin_src bash
bd create "Define request.Request type" -p 0
bd create "Implement CSP predicate" -p 1
bd dep <csp-id> --blocks-on <request-type-id>  # can't start CSP before types exist
bd ready  # only shows unblocked work
#+end_src

* Contracts to enforce from commit 1

#+begin_src go
// internal/request/request.go
// INVARIANT: Request is a value type. Headers map must be deep-copied
// before mutation. No other reference-type fields.

// internal/predicate/types.go
// INVARIANT: NamedPred.Type determines which function field is set:
//   TypeUniversal   → Fn != nil, ReqFn == nil, MultiFn == nil
//   TypeRelational  → ReqFn != nil, Fn == nil, MultiFn == nil
//   TypeSequential  → MultiFn != nil, Fn == nil, ReqFn == nil
// Enforced by ValidateGroup() at init time.

// internal/executor/executor.go
// INVARIANT: Caller MUST close ALL response bodies in Result.Responses.
// Not just Result.Response (which is Responses[0]).

// internal/oracle/oracle.go
// INVARIANT: Shrink produces a request where |shrunk| ≤ |original|
// for every dimension (headers, auth, origin, method, repeat).
#+end_src
