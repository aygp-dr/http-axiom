* What Worked Well

1. **CPRR-driven development**: Every P0 bug had a conjecture with
   falsification criteria. Haxgoat endpoints were designed to expose
   the exact false negative. Validation was deterministic.

2. **Worktree isolation**: Agents worked in git worktrees, preventing
   working tree corruption. Each agent had its own branch.

3. **Dependency-ordered merging**: New packages first (clean), then
   shared-file changes sequentially. This minimized conflict count.

4. **Structural investment paying off**: The round 3 predicate split
   eliminated all round 4 conflicts.

5. **L7 review as a forcing function**: The boundary review found the
   P0 shallow-copy bug, identified the type extraction need, and
   motivated the predicate split — all of which improved quality.

* What Could Be Better

1. **Test files always conflict**: Each agent creates its own test file
   or appends to a shared one. Solution: per-feature test files
   (=predicate_csp_test.go=, =predicate_hsts_test.go=) or a
   consolidation agent.

2. **aq is advisory, not enforcing**: File-level ownership detection
   doesn't prevent conflicts. Would need a reservation/lock protocol
   or structural file splitting.

3. **Worktree drift**: Agents fork from the main branch at launch time.
   Agents that take longer end up merging into a main that has moved.
   A rebase-before-merge protocol would help.

4. **CPRR validation gap**: 4/11 conjectures are still OPEN because
   they require =hax run= (which isn't wired yet) or live testing
   against haxgoat. The validation loop is incomplete for multi-request
   properties.

5. **No automated smoke after merge**: Each merge was manually verified
   with =go build= + =go test=. A post-merge hook running =make smoke=
   would catch integration issues faster.
